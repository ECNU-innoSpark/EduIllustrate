Abstract

Generating high-quality illustrated explanations for K-12 STEM problems requires both geometrically accurate diagrams and coherent step-by-step reasoning—a combination that existing LLM-based educational content systems have not addressed. We present EduIllustrate, an automated four-stage agentic pipeline that takes a K-12 STEM problem as input and produces a structured, diagram-rich explanation with programmatically rendered static diagrams via Manim. Our pipeline introduces a sequential anchoring strategy: Scene 1 undergoes full implementation planning to establish visual conventions (color scheme, label formatting, spatial layout), while subsequent scenes generate in parallel conditioned on Scene 1's code, enforcing cross-diagram consistency without global optimization.

To enable systematic evaluation, we construct a benchmark of 118 problems spanning three grade levels (elementary, middle, high school) and five subjects (mathematics, physics, chemistry, biology, geography), and propose an 8-dimension evaluation rubric grounded in multimedia learning theory covering both text quality and visual quality. Evaluation of five frontier LLMs shows Gemini 3.0 Pro Preview achieves the highest overall score (4.40/5.0), while Kimi-K2.5 offers the best cost-efficiency (4.07 at $0.15/problem). Workflow ablation confirms our sequential anchoring improves Visual Consistency by 12% over a fully parallel baseline at 17.5% lower cost. Human evaluation with 20 expert raters validates LLM-as-judge reliability for objective dimensions (ρ=0.89 for logical coherence, ρ=0.83 for diagram-problem alignment), while revealing limitations on subjective visual assessment.

How EduIllustrate Works?

We introduce EduIllustrate: a four-stage agentic pipeline that decomposes illustrated explanation generation into sequential stages addressing specific multimodal content creation challenges. The pipeline introduces a sequential anchoring strategy where Scene 1 undergoes full implementation planning to establish visual conventions, while subsequent scenes generate in parallel conditioned on Scene 1's code, enforcing cross-diagram consistency.


Figure 2: EduIllustrate pipeline overview. Scene 1 undergoes full sequential processing (outline → implementation planning → code generation); Scenes 2–N skip implementation planning and generate in parallel, each conditioned on Scene 1's code as a visual anchor.


Stage 1: Structured Outline Generation. Transforms raw K-12 STEM problems into structured outlines interleaving textual explanation blocks with visual scene specifications.

Stage 2: Scene Implementation Planning. Applies exclusively to Scene 1; Scenes 2–N skip this stage. Converts high-level Scene 1 specifications into detailed implementation plans, establishing visual conventions that all subsequent scenes inherit.

Stage 3: Manim Code Generation and Rendering. Scene 1 generates code conditioned on its implementation plan. Scenes 2–N generate in parallel, each receiving Scene 1's code as context to maintain visual consistency.

Stage 4: Markdown Assembly. Integrates rendered PNG diagrams with textual explanations into a structured Markdown document, with optional translation to Chinese.

Key Contributions

An automated four-stage pipeline for generating diagram-rich K-12 STEM explanations with explicit geometric accuracy and visual consistency constraints
A curated benchmark of 118 problems across 3 grade levels and 5 subjects for standardized multimodal evaluation
An 8-dimension evaluation rubric with automated and human protocols grounded in educational psychology
Comparative study of five frontier LLMs identifying model-specific strengths and failure modes
Validation of LLM-as-judge reliability for multimodal content through correlation analysis with human expert ratings
